3주차 백엔드 스터디에서는 JPA와 레포지토리를 중심으로 영속성과 트랜잭션 개념을 정리하였다.
JPA는 객체와 데이터베이스를 매핑하는 기술이고 이는 영속성 컨텍스트를 통해 엔티티의 상태를 관리한다는 점이 가장 중요하다. 
엔티티는 비영속, 영속, 준영속, 삭제 상태로 나뉘며 이러한 생명주기를 통해서 데이터가 어떤 시점에 관리되고 반영되는지를 구조적으로 파악하였다. 
또한 동일 트랜잭션 내에서 같은 엔티티를 조회하면 동일한 객체를 반환하는 "동일성 보장"과 1차 캐시 개념을 통해 성능과 일관성을 확보한다는 점을 학습하였다.

트랜잭션은 데이터의 일관성을 유지하기 위한 기본 단위로 JPA에서는 영속성 컨텍스트와 함께 동작하며 하나의 작업 흐름을 형성한다. 
트랜잭션과 관련해서는 @Transactional 어노테이션을 기준으로 트랜잭션의 시작과 종료 범위를 학습하였으며 하나의 트랜잭션 안에서 수행되는 작업들이 하나의 단위로 처리된다는 점을 실제 코드 흐름과 함께 이해하였다
트랜잭션 내에서 수행되는 작업들은 하나의 단위로 묶여 처리되고 문제가 발생할 경우 전체 작업이 취소된다는 점에서 안정적인 데이터 처리가 가능해진다. 
이러한 흐름을 이해하면서 단순한 DB 작업이 아니라 애플리케이션 전반의 데이터 처리 구조를 이해려고 노력헀다.

또한 지연 로딩을 통해 연관 엔티티를 즉시 조회하지 않고 실제로 사용하는 시점에 조회가 이루어진다는 점을 학습하였다. 컴퓨터 네트워크 시간에 배운 Store and Forward와 비슷한 방식이어서 더 크게 와닿았다.
이를 통해 불필요한 조인을 줄이고 성능을 개선할 수 있지만, 트랜잭션 범위를 벗어난 경우 프록시 초기화 문제가 발생할 수 있다는 점도 함께 다루었다. 
이러한 내용을 통해 지연 처리의 장점과 주의점 모두를 이해할 수 있었다.

레포지토리는 이러한 JPA 기능을 기반으로 데이터 접근 계층을 추상화하여 개발자가 비즈니스 로직에 집중할 수 있도록 돕는 역할을 한다.
인터페이스 기반 설계를 통해 구현을 단순화하고 코드의 유지보수성을 높일 수 있다는 점에서 실무적인 이득이 있을 것 같다고 생각했다.
